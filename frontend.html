Итак, план изучения фронтэнда по Сереге Дмитриевскому:

  1. Создать и подключить экосистему Гита: установить Git, изучить основные команды Git, создать репозиторий на GitHub, сделать Hello World на HTML, опубликовать страничку на GitHub Pages.
  2. Выбрать макет для верстки на Figma
  3. Реализовать структуру сайта: изучить теги HTML, изучить селекторы и правила CSS, изучить методологию БЭМ и правила именования классов, разбить макет на блоки, подключить reset.css, сверстать структуру и базовое отображение для десктопа, опубликовать на GitHub Pages.
  4. Интерактив сайта: Изучить основы JS, выбрать элементы на странице и модифицировать их, подключить на страницу интерактивные элементы - слайдер, попап и т.д.
  5. Соответствие сайта макету, корректность отображения во всех браузерах и устройствах: изучить медиа-queries, изучение резиновой верстки, проверка на pixel perfect, проверка на разных устройствах, на всех браузерах.
  6. Оптимизация работы сайта: оптимизация кода, изображений, автоматизация оптимизации, измерение эффективности работы сайта.
  7. Улучшение структуры кода: есть ли повторяющийся код в проекте?, легкость поддержки и доработки кода: шаблонизация, css препроцессоры, линтинг кода, автоматизация сборки, документация на код.
  8. Сделать 3 таких проекта в портфолио.

Основные команды Git:

  • git clone [адрес]: клонирование (скачивание) папки проекта из репозитория на локальную машину
  • git pull: синхронизация локальной папки с сервером
  • git checkout: проверка активной ветки
  • git checkout dev/master: переключение между ветвями
  • git status: проверка статуса изменений
  • git add [файл]: добавление файла в статус trackked
  • git commit -m 'comment': синхронизация изменений
  • git push: публикация изменений - запись измененных файлов в репозиторий
  • Справочник основных команд Git
